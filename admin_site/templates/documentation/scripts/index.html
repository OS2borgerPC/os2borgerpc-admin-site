{% translate "Administrating Scripts" %}

{% blocktranslate trimmed %} When you click on "Scripts" in the left-hand menu, a list of scripts will be shown to the right of the left-hand menu and the editing interface for scripts will be shown to the right of this list. {% endblocktranslate %}

{% blocktranslate trimmed %} Scripts are for individual actions whereas security scripts monitor special events. {% endblocktranslate %}

{% blocktranslate trimmed %} The list of scripts is divided in two parts that one can swap between by clicking on the tabs "Local scripts" and "Global scripts" above the list. {% endblocktranslate %}

{% blocktranslate trimmed %} Local scripts are scripts that can only be used on the currently chosen site while global scripts are scripts that can be used on all sites. {% endblocktranslate %}

{% translate "You have the following options for every script" %}: {% include 'documentation/scripts/__submenu__.html' %}

{% blocktranslate trimmed %} If you wish to add a new script, you must click on "Add new Script" above the list of scripts. This will open a modal where you can fill out the name and description and upload the script code. When you then click on "Create script", the script will be created, and you will be redirected to the editing interface for the newly created script where you can add arguments for the script. {% endblocktranslate %}
